= Locating
:description: Public entry page for the Locating component.
:page-slug: locating

Locating is a Symfony-oriented locator, normalization, validation, and geocoding component within the Smartresponsor ecosystem.

It should be understood not as a simple map helper or geocode lookup wrapper, but as a governed locator engine for address normalization, validation, provider routing, geocoding, caching, resilience, tenant-aware controls, and observability.

== What It Is

Locating exists to provide a structured internal location capability that can support address processing, validation, geocoding, provider orchestration, runtime resilience, and machine-facing locator operations.

In ecosystem terms, it belongs to the class of components that strengthen location correctness, address quality, and operational location intelligence.

== Current Reading

In its current state, Locating already reads as a serious pre-RC component.

Its strongest visible qualities are:

* clear locator-engine product identity,
* strong provider-aware and resilience-aware runtime posture,
* broad test coverage across unit, integration, contract, and smoke lanes,
* and a governance-oriented engineering posture around tenant-aware controls and observability.

== Why It Matters

Location systems become fragmented when normalization, validation, geocoding, provider selection, caching, resilience, and operational observability are treated as separate ad hoc concerns.

Locating matters because it treats those concerns as part of one governed component surface rather than as scattered utility logic.

That makes it strategically more valuable than a narrow geocode helper or a thin third-party location adapter.

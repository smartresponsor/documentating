= Documentating Cleanup Audit
:revdate: 2026-04-01

== Current state

The repository now uses one normal publication contract:

* publish owner: `.github/workflows/gh-pages.yml`
* builder entry: `tools/build_antora_site.py`
* playbook: `antora-playbook.yml`
* asset source: `assets/`

== Completed evacuation

The following legacy surface has been removed instead of being kept as transitional replay shells:

* versioned and owner-oriented workflows under `.github/workflows/`
* owner playbooks `antora-playbook-owner.yml` and `antora-playbook-owner-v2.yml`
* removed legacy wrapper builders `build_antora_site_v2.py`, `v5.py`, `v6.py`, `v7.py`, `v9.py`, `v13.py`, `v14.py`, `v15.py`, `v17.py`

== Canonical builder line

The normal-path builder surface has now been collapsed to one file:

* `tools/build_antora_site.py`

That file now directly owns:

* base Antora generation,
* owner-facing home page write,
* component page preservation/restore,
* slug override expansion,
* pre-RC Quality Atlas generation.

There is no remaining active multi-file builder composition in the canonical path.


== Legacy atlas generator removal

The repository now keeps one atlas generator in the normal path:

* `tools/generate_pre_rc_quality_atlas.py`

The older generator `tools/generate_quality_atlas.py` has been removed as legacy residue because the repository contract now targets the pre-RC atlas scope directly.

== Generated Index Residue Removal

The canonical builder already generated the public portal index surfaces directly into `.antora-src`, and the following repository markdown index residues were skipped by the builder:

* `docs/index.md`
* `docs/article/index.md`
* `docs/component/index.md`

These legacy markdown index files have now been removed to align the repository with the one-contract Antora-first path.


== Skipped-source residue evacuation

The legacy `docs/articles/` source surface has also been removed so that the canonical article source surface is now only `docs/article/`.
